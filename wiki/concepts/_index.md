# concepts/

Cross-cutting ideas, techniques, and design principles the repo's artifacts embody or teach — the
"why it works this way" knowledge that isn't tied to a single artifact.

**Page format:** lead with a one-line definition, then explain the idea and where it shows up across
the artifacts, with provenance (`[[links]]` to artifact pages, file paths, or external writeups). One
concept per page.

A concept earns a page when it recurs across more than one artifact, or when an artifact page would
otherwise have to re-explain it. If an idea is specific to a single artifact, keep it on that
artifact's page instead.
